## 核心设计模式

### 1. 分层架构 (Layered Architecture)

| 层级 | 目录 | 职责 |
|------|------|------|
| **应用层** | `app/` | 用户交互、控制逻辑、状态管理 |
| **核心层** | `core/` | 通用算法、坐标变换、数据结构 |
| **机器人层** | `robots/` | 机器人特定的参数和算法 |

**优势**:
- 层与层之间低耦合
- 便于替换机器人实现
- 核心算法可复用

---

### 2. 抽象基类 (Abstract Base Class)

```python
# core/kinematics/ik_base.py
class IKSolverBase(ABC):
    @abstractmethod
    def forward(self, hip_pos_body, joints):
        """正运动学：关节角度 → 关节位置"""
        raise NotImplementedError

    @abstractmethod
    def inverse(self, foot_pos_in_hip_frame):
        """逆运动学：脚部位置 → 关节角度"""
        raise NotImplementedError
```

**实现示例**:

### 4. 观察者模式 (Observer Pattern)

```python
# core/transform.py
class WorldTransform:
    def __init__(self):
        self._on_change = None  # 回调函数

    def _invalidate(self):
        self._T = None  # 清除缓存
        if self._on_change:
            self._on_change()  # 通知观察者

# app/robot_model.py
def _setup_frame_callbacks(self):
    for node in self.frame_manager.nodes.values():
        if node.rel:
            node.rel.set_on_change(self.invalidate_cache)
```

**作用**: 坐标系变化时自动清除相关缓存

---

### 5. 缓存模式 (Cache Pattern)

```python
# app/robot_model.py
class RobotModel:
    def __init__(self):
        self._cache_body_joints = {}   # 机体坐标系缓存
        self._cache_world_joints = {}  # 世界坐标系缓存

    def get_leg_joints_body(self, leg_name):
        if leg_name in self._cache_body_joints:
            return self._cache_body_joints[leg_name]  # 返回缓存
        # 计算并缓存
        result = self._compute_leg_joints(leg_name)
        self._cache_body_joints[leg_name] = result
        return result
```

**优势**: 避免重复的坐标变换计算，提高性能

## 数据流向分析

### 用户控制流程

```
用户操作 UI 滑块
       ↓
ui_matplotlib.py: update_robot()
       ↓
controller.py: set_body_pose() 或 set_joint_angles()
       ↓
   ┌─────────────────────────────────────┐
   │  判断控制模式                         │
   ├─────────────────────────────────────┤
   │  逆运动学模式 (foot_lock_enabled)    │
   │  → 捕获脚步位置                       │
   │  → 调用 IK 计算新关节角度             │
   │  → 更新模型                          │
   ├─────────────────────────────────────┤
   │  正向运动学模式 (!foot_lock_enabled) │
   │  → 直接设置关节角度                   │
   └─────────────────────────────────────┘
       ↓
robot_model.py: update_joint_angles()
       ↓
robot_model.py: invalidate_cache()  # 清除缓存
       ↓
ui_matplotlib.py: _draw_robot_in_world()  # 重新绘制
       ↓
显示更新后的机器人状态
```

### 坐标变换流程

```
关节角度 (LegJoints)
       ↓
leg_kinematics.py: forward()
       ↓
关节位置 (机体坐标系)
       ↓
frame_manager.py: transform_point()
       ↓
world → body → hip_*
       ↓
关节位置 (世界坐标系)
       ↓
UI 绘制
```

## 性能优化设计

### 1. 缓存机制

| 缓存类型 | 存储内容 | 失效条件 |
|---------|---------|---------|
| 机体坐标系关节位置 | `_cache_body_joints` | 关节角度更新 |
| 世界坐标系关节位置 | `_cache_world_joints` | 关节角度更新或坐标系变化 |
| 变换矩阵 | `_T` in WorldTransform | 位姿更新 |

### 2. 懒加载

```python
# transform.py
def matrix(self):
    self._build_matrix()  # 按需计算
    return self._T.copy()
```

### 3. 批量更新

```python
# 批量设置关节角度时，只清除一次缓存
def set_all_joint_angles(self, joint_angles_dict):
    self.invalidate_cache()  # 一次性清除所有缓存
    for leg_name, angles in joint_angles_dict.items():
        self._update_single_leg(leg_name, angles)
```

## 错误处理设计

### IK 计算失败处理

```python
class IKResult:
    success: bool              # 是否成功
    joints: Optional[LegJoints] # 计算结果
    reason: Optional[str]      # 失败原因

# 使用示例
ik_result = legkin.inverse(foot_pos)
if not ik_result.success:
    print(f"IK 失败: {ik_result.reason}")
    # 处理失败情况
```

### 常见错误类型

| 错误 | 原因 | 处理方式 |
|------|------|---------|
| Target too close to hip | 目标点太靠近髋关节 | 返回失败，不更新关节 |
| Target too far | 目标点超出工作空间 | 返回失败，保持原位置 |
| Math error | 数值计算错误 | 返回失败，打印错误信息 |
