## 问题诊断与修复

### 问题1：初始姿态错误（P0）

**现象：**
- IK成功率：20-40%
- 腿完全伸直（零位姿态）
- 步态轨迹超出工作空间

**原因：**
```python
# ❌ 错误：零位姿态（腿完全伸直）
initial_joints = LegJoints(0.0, 0.0, 0.0)
```

**解决方案：**
```python
# ✅ 正确：站立姿态（膝关节弯曲）
hip_pitch_deg = 15.0
knee_deg = -45.0  # 膝关节弯曲45°

for leg_name in model.legs.keys():
    joints = LegJoints(
        0.0,  # hip_side
        math.radians(hip_pitch_deg),  # hip_pitch
        math.radians(knee_deg)  # knee_pitch
    )
    model.update_joint_angles(leg_name, joints)
```

**效果：**
- IK成功率：35% → 100%

---

### 问题2：足端轨迹理解错误（P0）

**错误理解：**
```python
# ❌ 错误：将步态偏移当作绝对位置
x, z = gait.get_foot_trajectory(leg_name)
foot_pos_hip = np.array([x, 0, z])  # 直接当作坐标
```

**正确理解：**
```python
# ✅ 正确：初始位置 + 步态偏移
initial_pos = initial_foot_positions[leg_name]  # 站立姿态下的初始位置
x_offset, z_offset = gait.get_foot_trajectory(leg_name)  # 步态偏移量
target_pos = initial_pos + np.array([x_offset, 0, z_offset])
```

**关键概念：**
- 步态轨迹（`x_offset`, `z_offset`）是**相对偏移量**
- 必须在**站立姿态**下获取初始足端位置
- 目标位置 = 初始位置 + 偏移量

### 问题3：机体坐标系未更新（P1）

**现象：**
- 机体原地踏步
- 没有连续前进

**错误代码：**
```python
# ❌ 错误：相位重置时机体回退
body_x = gait.global_phase * gait.stride_length
# 相位从0→1后重置为0，机体位置也回退
```

**正确代码：**
```python
# ✅ 正确：累计前进距离
total_distance = 0.0
previous_phase = 0.0

def update():
    global total_distance, previous_phase
    
    gait.update(dt)
    
    # 检测相位重置（新周期开始）
    if gait.global_phase < previous_phase:
        total_distance += gait.stride_length
    
    previous_phase = gait.global_phase
    
    # 计算当前机体位置
    body_x = total_distance + gait.global_phase * gait.stride_length
    controller.set_body_pose(body_x, 0, -0.1, 0, 0, 0)
```

**效果：**
- 机体连续前进，不回退
- 3个周期前进12cm（4cm/周期）

### 问题4：占空比错误（P0 - 最关键）

**现象：**
- 摆动相占50%，支撑相占50%
- 某时刻只有2条腿支撑
- 不符合Walk步态标准

**错误代码：**
```python
# app/gait/trajectory.py
def cycloid_trajectory(phase, stride_length, step_height):
    if phase < 0.5:  # ❌ 摆动相占50%
        t = phase * 2 * np.pi
        x = stride_length * (1 - t / (2*np.pi) + np.sin(t) / (2*np.pi))
        z = step_height * (1 - np.cos(t)) / 2
    else:  # 支撑相占50%
        x = stride_length * (1 - phase)
        z = 0
    return x, z
```

**正确代码：**
```python
def cycloid_trajectory(phase, stride_length, step_height):
    if phase < 0.25:  # ✅ 摆动相占25%
        t = phase * 4 * np.pi  # 从2改为4
        x = stride_length / 2 * (t / np.pi - 1)
        z = step_height * (1 - np.cos(t)) / 2
    else:  # ✅ 支撑相占75%
        t = (phase - 0.25) * 4 / 3 * np.pi
        x = stride_length / 2 * (1 - t / np.pi)
        z = 0
    return x, z
```

**验证：**
```python
# 验证脚本
for phase in [0.0, 0.25, 0.5, 0.75, 1.0]:
    x, z = cycloid_trajectory(phase, 0.04, 0.025)
    state = '摆动' if phase < 0.25 else '支撑'
    print(f'相位{phase:.2f} ({state}): x={x*100:+5.2f}cm, z={z*100:+5.2f}cm')
```

**效果：**
- 占空比：25%/75%（符合标准）
- 每时刻3足支撑（稳定）

## Walk步态标准与实现

### Walk步态定义

**特点：**
- ✅ 占空比：摆动25%，支撑75%
- ✅ 每时刻恒定3条腿支撑
- ✅ 相位差：90°（依次抬腿）
- ✅ 稳定性：极高（三角形支撑面）

**相位安排：**

| 腿 | 相位偏移 | 角度 | 抬腿顺序 |
|---|---|---|---|
| 右前腿 | 0.00 | 0° | 第1个抬起 |
| 左后腿 | 0.25 | 90° | 第2个抬起 |
| 左前腿 | 0.50 | 180° | 第3个抬起 |
| 右后腿 | 0.75 | 270° | 第4个抬起 |

**一个完整周期（0→1）：**
```
相位 0.00-0.25: 右前腿抬起，其他3腿支撑
相位 0.25-0.50: 左后腿抬起，其他3腿支撑
相位 0.50-0.75: 左前腿抬起，其他3腿支撑
相位 0.75-1.00: 右后腿抬起，其他3腿支撑
```

### 实现

**步态控制器：**
```python
from app.gait.walk_gait import WalkGait

gait = WalkGait(
    stride_length=0.04,  # 步长4cm

    frequency=0.8        # 步频0.8Hz
)

# 更新步态
```

## 关键经验总结

### 1. 初始姿态的重要性

**经验：**
- 零位姿态（腿伸直）会导致IK失败
- 必须在站立姿态（膝关节弯曲）下获取初始足端位置
- 膝关节弯曲角度：45°

**代码：**
```python
def set_standing_pose(model):
    hip_pitch_deg = 15.0
    knee_deg = -45.0
    
    for leg_name in model.legs.keys():
        joints = LegJoints(0.0, math.radians(hip_pitch_deg), math.radians(knee_deg))
        model.update_joint_angles(leg_name, joints)
```

### 2. 步态偏移的正确理解

**概念：**
- 步态轨迹是**相对偏移量**，不是绝对位置
- 目标位置 = 初始位置 + 偏移量
- 初始位置必须在站立姿态下获取

**代码：**
```python
# 1. 捕获初始足端位置（站立姿态）
initial_positions = capture_initial_positions(model)

# 2. 在动画循环中：初始位置 + 偏移
for leg_name, (x_offset, z_offset) in trajectories.items():
    initial_pos = initial_positions[leg_name]
    target_pos = initial_pos + np.array([x_offset, 0, z_offset])
    ik_result = legkin.inverse(target_pos)
```
