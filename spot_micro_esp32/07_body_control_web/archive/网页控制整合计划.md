# ESP32 Web 端与机器人控制整合实施计划（修订版 v1.2）

目标：在现有工程 spot_micro_esp32/body_control_web 内，直接整合 simple 的 HTTP 网页控制界面，实现网页端对四足机器人的实时控制（平滑姿态 pose6 与连续动作 action_cy）。采用最小改动策略，避免创建新工程。

日期：2025-09-26

------------------------------------------------------------
一、现状与对接点梳理
------------------------------------------------------------

1) Web 控制端（simple/main.c）
- HTTP 路由：
  - /api/pos/x, /api/pos/y, /api/pos/z
  - /api/rot/roll, /api/rot/pitch, /api/rot/yaw
  - /api/action/1..5
- 前端：HTML+JS 滑块（200ms 防抖），按钮触发 /api/action/1。

2) 机器人控制栈（1_pca/main/pca9685Test.cpp + controllers/*）
- 已实现：
  - init_robot_system() 初始化 RobotController 与 Kinematics
  - initSmoothMotionSystem() 初始化 SmoothMotionController 与 QuadrupedModel
  - pose6 x y z r p y：平滑姿态到目标；位置单位米、姿态单位度（内部转弧度）
  - action_cy [beta]：圆锥连续动作（起始位姿 → 36 点轨迹 → 回中性位姿）

结论：将 simple 的 HTTP 控制映射到 pca 的平滑控制接口（pose6/action_cy），并确保先 smooth_init。

二、整合策略（最小改动）
------------------------------------------------------------

- 不新建 web_robot 工程；直接在 1_pca 工程 main/ 目录增加两个文件：

  2) web_server_adapter.c：轻量 HTTP 服务器，复用 simple 的路由与 HTML，调用 robot_bridge_* 接口

- 保留 pca 的 UART 命令行（parseCommand），与网页控制共存；网页控制优先。

四、指令映射与单位换算
------------------------------------------------------------

- 滑块值范围与映射（服务器侧统一换算与限幅）：
  - X: [-100..100] → x(m) = value * 0.001 → [-0.10..0.10]
  - Y: [-100..100] → y(m) = value * 0.001 → [-0.10..0.10]
  - Z: [-100..100] → z(m) = -0.10 + value * (-0.001) → 约 [-0.20..0.0]
    - 实际安全限幅：z ∈ [-0.22..-0.05]（桥接层二次限幅）
  - roll/pitch/yaw: [-180..180]（度）→ 内部统一转弧度

- 状态聚合：
  - pose_state = {x,y,z,roll_deg,pitch_deg,yaw_deg}
  - 每次滑块更新单轴后，调用 robot_bridge_pose6(pose_state 六参)，覆盖最新目标。

- 动作：
  - /api/action/1?beta=30 → robot_bridge_action_cone(30)
  - beta 度数限幅 [5..45]；越界返回 400

------------------------------------------------------------
五、桥接层接口定义（main/robot_bridge.h/.cpp）
------------------------------------------------------------

- 头文件 robot_bridge.h
  - int robot_bridge_init(void);
    - 调用 init_robot_system() 与 initSmoothMotionSystem()；返回 0 成功，负值为错误码
  - int robot_bridge_pose6(float x, float y, float z,
                           float roll_deg, float pitch_deg, float yaw_deg);
    - 角度转弧度，SmoothMotionController::setTargetPose → executeMotionToTarget；0 成功
  - int robot_bridge_action_cone(float beta_deg);
    - 复制 action_cy 逻辑（起始位姿 → 36 点轨迹 → 回中性位姿）；0 成功

- 可选低层接口（后续扩展）：
  - int robot_bridge_set_joint_angle(int leg, int joint, float angle_deg);
  - int robot_bridge_set_servo_angle(int servo_id, float angle_deg);
  - int robot_bridge_set_pwm(int channel, uint16_t pwm);

- 互斥与并发：
  - 在 robot_bridge.cpp 内部使用 FreeRTOS mutex（或任务队列）保证 setTargetPose/executeMotionToTarget 串行
  - 新目标到来时采用“覆盖最新目标”，必要时停止当前运动，优先执行最新 pose6

- 错误码约定：
  - 0=OK；-EINVAL 参数错误；-EIO 驱动错误；-EBUSY 控制器忙；-EPERM 未初始化

- 伪代码示例（robot_bridge_pose6）：
  ```
  int robot_bridge_pose6(float x, float y, float z, float roll_deg, float pitch_deg, float yaw_deg) {
      if (!g_smooth_motion_initialized_) return -EPERM;
      // 限幅
      x = clamp(x, -0.12f, 0.12f);
      y = clamp(y, -0.12f, 0.12f);
      z = clamp(z, -0.22f, -0.05f);
      // 角度→弧度
      float roll = roll_deg * M_PI / 180.0f;
      float pitch = pitch_deg * M_PI / 180.0f;
      float yaw = yaw_deg * M_PI / 180.0f;
      // 串行执行
      lock(mutex);
      g_smooth_controller_->setTargetPose(BodyPose(x,y,z,roll,pitch,yaw));
      bool ok = g_smooth_controller_->executeMotionToTarget();
      unlock(mutex);
      return ok ? 0 : -EIO;
  }
  ```

六、HTTP 适配层（main/web_server_adapter.c）
------------------------------------------------------------

- 路由与处理器：
  - /api/pos/x,y,z 与 /api/rot/roll,pitch,yaw → 解析 ?value=，范围校验与映射，更新 pose_state 并调用 robot_bridge_pose6()
  - /api/action/1 → 可解析 ?beta=，调用 robot_bridge_action_cone()
  - / → 返回 html_page（可直接复制 simple/main.c 中的 html_page）

- 返回 JSON：
  - 成功：{"status":"ok"}
  - 错误：{"status":"error","message":"..."} 并设置 400

- 启动流程（接口）：
  - httpd_handle_t start_webserver(void);  // 启动并注册路由
  - void stop_webserver(httpd_handle_t server); // 需要时关闭

- 伪代码示例（pos/rot 统一处理）：
  ```
  static PoseState g_pose = {0.05f, 0.0f, -0.10f, 0, 0, 0};
  esp_err_t unified_api_handler(httpd_req_t *req) {
      const char* uri = req->uri; // e.g., /api/pos/x?value=50
      int v = parse_query_value(req, "value");
      if (is_pos_axis(uri, 'x')) g_pose.x = v * 0.001f;
      else if (is_pos_axis(uri, 'y')) g_pose.y = v * 0.001f;
      else if (is_pos_axis(uri, 'z')) g_pose.z = -0.10f + v * (-0.001f);
      else if (is_rot_axis(uri, "roll"))  g_pose.roll = v;
      else if (is_rot_axis(uri, "pitch")) g_pose.pitch = v;
      else if (is_rot_axis(uri, "yaw"))   g_pose.yaw = v;
      int rc = robot_bridge_pose6(g_pose.x, g_pose.y, g_pose.z,
                                  g_pose.roll, g_pose.pitch, g_pose.yaw);
      return rc==0 ? send_ok(req) : send_err(req, rc);
  }
  ```

- 网络参数：
  - 采用 simple/main.c 的配置（端口、lru_purge、超时等），保持稳定
  - 保留精确匹配 URI 注册，避免通配符依赖

十、风险与注意事项
------------------------------------------------------------

- 硬件限幅：桥接层与 HTTP 层双重限幅，避免触发机构极限
- 并发控制：严格串行控制，防止 pose6 重入；必要时采用控制队列与“最后写胜出”策略
- 资源占用：HTML 常量占 RAM；后续可迁移至 SPIFFS 或分块发送
- 日志级别：动作过程中降低高频日志，保留必要告警与错误
- 单位一致性：服务器层统一使用“米”和“度”（内部转换为弧度）
