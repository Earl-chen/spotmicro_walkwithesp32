### 核心模块说明

#### 1. 网页客户端层
- **Material Design界面**: 现代化深蓝色科技风格
- **实时控制面板**: 位置、姿态6自由度滑块控制
- **预定义动作区**: 一键执行复杂动作序列
- **状态监控显示**: 实时显示机器人状态信息
- **双协议支持**: WebSocket优先，HTTP降级备用

#### 2. ESP32通信层
- **HTTP服务器**: 处理RESTful API请求
- **WebSocket服务**: 提供低延迟实时通信
- **桥接适配层**: 网页指令到机器人控制的无缝转换
- **参数验证**: 确保控制指令的安全性和有效性

#### 3. 机器人控制层
- **运动学计算**: 6自由度姿态到关节角度转换
- **平滑运动控制**: 消除运动抖动，提供流畅体验
- **预定义动作库**: 圆锥运动等复杂动作序列
- **安全保护机制**: 紧急停止和参数范围限制

#### 4. 硬件执行层
- **PCA9685 PWM驱动**: 16通道高精度PWM输出
- **12路舵机控制**: 四足机器人3×4关节精确控制
- **I2C通信协议**: 稳定可靠的硬件通信
- **5V电源管理**: 为舵机提供充足动力

## 🚀 核心技术创新

### 1. 双协议通信架构
- **WebSocket优先**: 实现10-30ms超低延迟实时控制
- **HTTP降级备用**: 确保在网络不稳定时的可用性
- **自动切换机制**: 智能选择最优通信方式

### 2. 多层次桥接设计
- **参数映射层**: 网页参数到机器人坐标系的无缝转换
- **指令验证层**: 确保所有控制指令的安全性
- **执行缓冲层**: 平滑处理高频控制指令

### 3. 嵌入式Web界面
- **单文件部署**: 网页界面完全嵌入ESP32固件
- **离线可用**: 无需外部Web服务器支持
- **响应式设计**: 支持手机、平板、电脑多端访问

### 4. 实时状态反馈
- **双向通信**: 不仅发送控制指令，还接收状态反馈
- **心跳监控**: 实时监测连接状态和系统健康
- **异常处理**: 自动重连和错误恢复机制

## 🔧 部署和配置

### 硬件要求
- **ESP32开发板**: ESP32-DevKitC或兼容板
- **PWM驱动器**: PCA9685 16通道PWM驱动板
- **舵机**: SG90或MG996R舵机 × 12个
- **电源**: 5V/3A电源适配器
- **连接线**: 杜邦线、面包板等

### 软件环境
- **ESP-IDF**: v4.4或更高版本
- **编译工具**: GCC交叉编译工具链
- **开发环境**: VS Code + ESP-IDF插件(推荐)

### 快速部署步骤

1. **环境准备**
```bash
git clone https://github.com/espressif/esp-idf.git
cd esp-idf
./install.sh
. ./export.sh
```

2. **项目编译**
```bash
cd spot_micro_esp32/body_control_web
idf.py menuconfig  # 配置WiFi信息
idf.py build
```

3. **固件烧录**
```bash
idf.py -p /dev/ttyUSB0 flash monitor
```

4. **网页访问**
```
打开浏览器访问: http://192.168.1.7:12580
```
