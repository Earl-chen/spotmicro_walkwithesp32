## 📡 网络连接问题

### 问题1: WiFi连接失败

**症状**:
```
E (1234) wifi: WiFi连接失败
E (1235) wifi: 认证超时
```

**可能原因**:
- WiFi密码错误
- 信号强度不足
- 路由器设置问题
- ESP32硬件问题

**解决方案**:

1. **检查WiFi配置**:
```c
// 在main/web_server_adapter.c中确认配置
#define WIFI_SSID "your_wifi_ssid"
#define WIFI_PASS "your_wifi_password"
```

2. **检查信号强度**:
```bash
# 在路由器附近测试
idf.py monitor
# 查看RSSI值，应该 > -70dBm
```

3. **重置网络设置**:
```c
// 添加到初始化代码
esp_wifi_restore();
nvs_flash_erase();
nvs_flash_init();
```

4. **使用手机热点测试**:
```c
// 临时修改为手机热点配置
#define WIFI_SSID "Phone_Hotspot"
#define WIFI_PASS "hotspot_password"
```

### 问题2: IP地址获取失败

**症状**:
```
I (5000) wifi: WiFi连接成功
E (10000) wifi: 获取IP地址超时
```

**解决方案**:

1. **检查DHCP设置**:
```c
// 确保启用DHCP客户端
esp_netif_dhcpc_start(esp_netif_get_handle_from_ifkey("WIFI_STA_DEF"));
```

2. **设置静态IP**:
```c
esp_netif_ip_info_t ip_info;
IP4_ADDR(&ip_info.ip, 192, 168, 1, 100);
IP4_ADDR(&ip_info.gw, 192, 168, 1, 1);
IP4_ADDR(&ip_info.netmask, 255, 255, 255, 0);
esp_netif_set_ip_info(netif, &ip_info);
```

3. **重启路由器**:
```bash
# 重启路由器并清除DHCP租约
# 等待2分钟后重新测试
```

### 问题3: 网络断线重连

**症状**:
```
W (30000) wifi: 网络连接丢失
I (35000) wifi: 正在重连...
```

**解决方案**:

1. **实现自动重连**:
```c
static void wifi_event_handler(void* arg, esp_event_base_t event_base,
                              int32_t event_id, void* event_data) {
    if (event_base == WIFI_EVENT && event_id == WIFI_EVENT_STA_DISCONNECTED) {
        ESP_LOGI(TAG, "WiFi断开，正在重连...");
        esp_wifi_connect();
        xEventGroupClearBits(s_wifi_event_group, WIFI_CONNECTED_BIT);
    }
}
```

2. **增加重连延迟**:
```c
static int retry_count = 0;
if (retry_count < 10) {
    vTaskDelay(pdMS_TO_TICKS(1000 * (retry_count + 1)));
    esp_wifi_connect();
    retry_count++;
}
```

## 🔗 WebSocket连接问题

### 问题7: WebSocket连接失败

**症状**:
```javascript
WebSocket connection to 'ws://192.168.1.7:12580/ws' failed
```

**解决方案**:

1. **检查WebSocket端点**:
```javascript
// 确保使用正确的协议和端点
const ws = new WebSocket('ws://192.168.1.7:12580/ws');
// 不是 wss:// (安全WebSocket)
```

2. **检查浏览器支持**:
```javascript
if (!window.WebSocket) {
    console.error('浏览器不支持WebSocket');
    // 降级到HTTP轮询
}
```

3. **增加连接超时**:
```javascript
const ws = new WebSocket('ws://192.168.1.7:12580/ws');
setTimeout(() => {
    if (ws.readyState !== WebSocket.OPEN) {
        console.error('WebSocket连接超时');
        ws.close();
    }
}, 10000);  // 10秒超时
```

### 问题8: WebSocket消息发送失败

**症状**:
```javascript
Failed to execute 'send' on 'WebSocket': Still in CONNECTING state
```

**解决方案**:

1. **等待连接建立**:
```javascript
const ws = new WebSocket('ws://192.168.1.7:12580/ws');

ws.onopen = function() {
    // 连接建立后再发送消息
    const message = {
        type: 'pose6',
        data: {x: 0.05, y: 0.0, z: -0.10, roll: 0.0, pitch: 0.0, yaw: 0.0},
        timestamp: Date.now()
    };
    ws.send(JSON.stringify(message));
};
```

2. **检查连接状态**:
```javascript
function sendMessage(message) {
    if (ws.readyState === WebSocket.OPEN) {
        ws.send(JSON.stringify(message));
    } else {
        console.error('WebSocket未连接');
        // 可以将消息加入队列，连接后发送
    }
}
```

### 问题9: WebSocket频繁断开

**症状**:
```javascript
WebSocket connection closed: code=1006
```

**解决方案**:

1. **实现心跳机制**:
```javascript
let heartbeatInterval;

ws.onopen = function() {
    // 每30秒发送心跳
    heartbeatInterval = setInterval(() => {
        if (ws.readyState === WebSocket.OPEN) {
            ws.send(JSON.stringify({
                type: 'heartbeat',
                data: {},
                timestamp: Date.now()
            }));
        }
    }, 30000);
};

ws.onclose = function() {
    clearInterval(heartbeatInterval);
};
```

2. **增加ESP32端缓冲区**:
```c
// 在websocket_handler.c中
#define WEBSOCKET_BUFFER_SIZE 1024  // 增加缓冲区大小
```

3. **实现自动重连**:
```javascript
function connectWebSocket() {
    const ws = new WebSocket('ws://192.168.1.7:12580/ws');
    
    ws.onclose = function(event) {
        console.log('WebSocket关闭，3秒后重连...');
        setTimeout(connectWebSocket, 3000);
    };
    
    return ws;
}
```

## 🤖 机器人控制问题

### 问题10: 舵机不响应

**症状**:
```
I (3000) robot_bridge: 发送姿态指令
W (3001) servo: 舵机无响应
```

**解决方案**:

1. **检查硬件连接**:
```
ESP32 -> PCA9685
GPIO21 (SDA) -> SDA
GPIO22 (SCL) -> SCL
3.3V -> VCC
GND -> GND

PCA9685 -> 舵机
PWM0-11 -> 舵机信号线
V+ -> 舵机电源正极 (5V)
GND -> 舵机电源负极
```

2. **检查I2C通信**:
```c
// 在drivers/PCA9685Driver.cpp中添加调试
esp_err_t ret = i2c_master_write_to_device(I2C_MASTER_NUM, PCA9685_ADDR, 
                                          write_buf, 2, I2C_MASTER_TIMEOUT_MS / portTICK_PERIOD_MS);
if (ret != ESP_OK) {
    ESP_LOGE(TAG, "I2C写入失败: %s", esp_err_to_name(ret));
}
```

3. **检查电源供应**:
```bash
# 使用万用表测量
# PCA9685 VCC: 应该是3.3V
# 舵机电源: 应该是5V，电流足够 (至少2A)
```

4. **测试单个舵机**:
```c
// 在pca9685Test.cpp中添加单舵机测试
void test_single_servo() {
    ServoDriver servo;
    servo.setAngle(0, 90);  // 设置0号舵机到90度
    vTaskDelay(pdMS_TO_TICKS(1000));
}
```

### 问题11: 运动不平滑

**症状**:
```
机器人运动时出现抖动或突然停止
```

**解决方案**:

1. **调整平滑参数**:
```c
// 在config/SmoothMotionConfig.hpp中
static constexpr float SMOOTH_FACTOR = 0.1f;  // 减小平滑因子
static constexpr int INTERPOLATION_STEPS = 20;  // 增加插值步数
```

2. **检查控制频率**:
```c
// 在controllers/SmoothMotionController.cpp中
static constexpr int CONTROL_FREQUENCY = 50;  // 50Hz控制频率
vTaskDelay(pdMS_TO_TICKS(1000 / CONTROL_FREQUENCY));
```

3. **优化运动轨迹**:
```c
// 使用更平滑的插值算法
float smoothStep(float t) {
    return t * t * (3.0f - 2.0f * t);  // 平滑步进函数
}
```

### 问题12: 姿态控制不准确

**症状**:
```
设置roll=30度，实际倾斜角度不正确
```

**解决方案**:

1. **校准舵机中位**:
```c
// 在config/ServoConfig.hpp中调整
struct ServoConfig {
    int channel;
    int min_pulse;    // 最小脉宽 (通常500-1000)
    int max_pulse;    // 最大脉宽 (通常2000-2500)
    int center_pulse; // 中位脉宽 (通常1500)
    bool reverse;     // 是否反向
};
```

2. **检查运动学参数**:
```c
// 在config/RobotConfig.hpp中确认
static constexpr float BODY_LENGTH = 0.18f;   // 机身长度 (m)
static constexpr float BODY_WIDTH = 0.08f;    // 机身宽度 (m)
static constexpr float LEG_LENGTH_1 = 0.055f; // 大腿长度 (m)
static constexpr float LEG_LENGTH_2 = 0.077f; // 小腿长度 (m)
```

3. **验证坐标变换**:
```c
// 在kinematics/CoordinateTransform.cpp中添加调试
ESP_LOGI(TAG, "输入姿态: roll=%.2f, pitch=%.2f, yaw=%.2f",

ESP_LOGI(TAG, "输出关节角度: [%.2f, %.2f, %.2f]",
```

## 💾 内存和性能问题

### 问题13: 内存不足

**症状**:
```
E (5000) heap: 堆内存不足
E (5001) task: 任务创建失败
```

**解决方案**:

1. **检查内存使用**:
```c
// 添加内存监控
void print_memory_info() {
    ESP_LOGI(TAG, "空闲堆内存: %d bytes", esp_get_free_heap_size());
    ESP_LOGI(TAG, "最小空闲堆内存: %d bytes", esp_get_minimum_free_heap_size());
}
```

2. **优化任务堆栈**:
```c
// 减少任务堆栈大小
xTaskCreate(web_server_task, "web_server", 4096, NULL, 5, NULL);  // 从8192减少到4096
```

3. **释放未使用的组件**:
```c
// 在CMakeLists.txt中移除不需要的组件
set(EXCLUDE_COMPONENTS "bt" "esp_adc_cal" "esp_eth")
```

4. **使用PSRAM**:
```bash
# 在menuconfig中启用PSRAM
idf.py menuconfig
# Component config -> ESP32-specific -> Support for external, SPI-connected RAM
```

### 问题14: CPU负载过高

**症状**:
```
系统响应缓慢，控制延迟增加
```

**解决方案**:

1. **优化任务优先级**:
```c
// 调整任务优先级
#define WEB_SERVER_TASK_PRIORITY 5      // 降低Web服务器优先级
#define ROBOT_CONTROL_TASK_PRIORITY 10  // 提高机器人控制优先级
```

2. **减少日志输出**:
```c
// 在生产环境中降低日志级别
esp_log_level_set("*", ESP_LOG_WARN);  // 只显示警告和错误
```

3. **优化控制算法**:
```c
// 使用查找表代替实时计算
static const float sin_table[360] = { /* 预计算的正弦值 */ };
float fast_sin(int angle) {
    return sin_table[angle % 360];
}
```
