## 🔧 核心模块详解

### 1. 桥接层 (robot_bridge)

桥接层是系统的核心组件，负责将网页控制指令转换为机器人控制参数。

#### 主要功能
- **指令映射**：将网页参数映射到机器人控制参数
- **参数转换**：单位转换和范围限制
- **状态管理**：维护机器人当前状态
- **错误处理**：统一的错误处理和日志记录

#### 接口设计
```cpp
// 六自由度姿态控制
int robot_bridge_pose6(float x, float y, float z, 
                      float roll, float pitch, float yaw);

// 圆锥动作执行
int robot_bridge_action_cone(float beta_deg);

// 预设姿势切换
int robot_bridge_set_preset_pose(int preset_id);

// 紧急停止
int robot_bridge_emergency_stop(void);

// 错误信息获取
const char* robot_bridge_get_last_error(void);
```

#### 参数映射规则
```cpp
// 位置映射：网页滑块 -> 机器人坐标
X轴: [-100, 100] mm -> [-0.100, 0.100] m
Y轴: [-100, 100] mm -> [-0.100, 0.100] m  
Z轴: [-100, 100] mm -> [-0.200, 0.000] m (相对于-0.10m基准)

// 姿态映射：网页滑块 -> 机器人角度
Roll:  [-180, 180] ° -> [-180, 180] °
Pitch: [-180, 180] ° -> [-180, 180] °
Yaw:   [-180, 180] ° -> [-180, 180] °
```

### 2. HTTP服务器适配层 (web_server_adapter)

HTTP适配层处理网页请求，提供RESTful API接口。

#### 路由设计
```
GET /                    - 主页面
GET /api/pos/x?value=N   - X轴位置控制
GET /api/pos/y?value=N   - Y轴位置控制  
GET /api/pos/z?value=N   - Z轴位置控制
GET /api/rot/roll?value=N   - Roll角度控制
GET /api/rot/pitch?value=N  - Pitch角度控制
GET /api/rot/yaw?value=N    - Yaw角度控制
GET /api/action/1?beta=N    - 圆锥动作
GET /api/preset/0           - 趴下姿势
GET /api/preset/1           - 站立姿势
GET /api/emergency/stop     - 紧急停止
```

#### 请求处理流程
```
1. 接收HTTP请求
2. 解析URI和查询参数
3. 参数验证和范围检查
4. 调用桥接层接口
5. 格式化响应结果
6. 发送JSON响应
```

### 3. WebSocket处理器 (websocket_handler)

WebSocket处理器提供实时双向通信能力。

#### 消息类型定义
```cpp
typedef enum {
    WS_MSG_POSE6,      // 姿态控制
    WS_MSG_ACTION,     // 动作执行
    WS_MSG_PRESET,     // 预设姿势
    WS_MSG_EMERGENCY,  // 紧急停止
    WS_MSG_HEARTBEAT,  // 心跳消息
    WS_MSG_STATUS      // 状态广播
} ws_message_type_t;
```

#### 连接管理
```cpp
// 连接状态跟踪
typedef struct {
    int fd;                    // 连接文件描述符
    bool is_active;           // 连接状态
    uint32_t last_heartbeat;  // 最后心跳时间
    uint32_t message_count;   // 消息计数
} ws_connection_t;
```

#### 心跳机制
- **客户端心跳**：每30秒发送心跳消息
- **服务端检测**：检测连接超时（60秒）
- **自动清理**：清理无效连接

## 🔄 数据流设计

### 控制指令流
```
网页滑块变化 → JavaScript事件 → WebSocket消息 → 
服务器解析 → 桥接层转换 → 机器人控制 → 硬件执行
```

### 状态反馈流
```
硬件状态 → 控制器状态 → 桥接层状态 → 
WebSocket广播 → 网页更新 → 用户反馈
```

### 错误处理流
```
硬件错误 → 驱动层检测 → 控制层处理 → 
桥接层记录 → WebSocket通知 → 网页显示
```

## ⚡ 性能优化设计

### 1. 通信优化
- **WebSocket优先**：减少HTTP握手开销
- **消息合并**：批量处理相似控制指令
- **防抖机制**：避免频繁的滑块更新
- **心跳优化**：30秒心跳间隔平衡实时性和资源消耗

### 2. 内存优化
- **静态分配**：避免动态内存分配
- **缓冲池**：复用消息缓冲区
- **连接限制**：最大8个并发连接

### 3. 实时性优化
- **中断驱动**：I2C通信使用中断模式
- **任务优先级**：WebSocket任务高优先级
- **无阻塞IO**：非阻塞网络操作

## 🛡️ 安全设计

### 1. 输入验证
```cpp
// 参数范围检查
if (value < -100 || value > 100) {
    return ROBOT_BRIDGE_ERROR_PARAM;
}

// 类型验证
if (!is_valid_float(param_str)) {
    return ESP_ERR_INVALID_ARG;
}
```

### 2. 连接安全
- **连接数限制**：防止DoS攻击
- **超时机制**：自动清理无效连接
- **错误计数**：异常连接检测

### 3. 硬件保护
- **参数限幅**：防止超出安全范围
- **紧急停止**：快速停止机制
- **状态检查**：硬件状态监控
